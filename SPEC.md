# 📘 **授業専用AIチャットシステム仕様書**

## 1. システム概要
本システムは、専門学校の授業において学生が生成AIを安全に利用できるようにするための **ログイン不要・ブラウザアクセスのみで利用可能な ChatGPT風チャットアプリケーション**である。

学生は共有PCからアクセスし、個人アカウント（SNSログイン等）を必要としない。  
APIキーはサーバー側で安全に管理し、フロントエンドには露出しない。

---

## 2. 目的
- 学生が生成AIのプロンプトを試せる環境を提供する  
- 共有PCでも安全に利用できる  
- ChatGPTに近いUI/UXを再現し、学習のハードルを下げる  
- 授業テーマに応じたプロンプト実験を行いやすくする  
- 必要に応じてログを収集し、授業改善に活用できる

---

## 3. システム構成（アーキテクチャ）

```
[学生PC]（ブラウザ）
      ↓ HTTP(S)
[フロントエンド]（静的HTML/JS）
      ↓ fetch()
[バックエンドAPI]（Node.js or Python）
      ↓
[LLM API]（OpenAI / Azure OpenAI / Gemini）
```

### 3.1 フロントエンド
- HTML + CSS + JavaScript（フレームワーク不要）
- ChatGPT風のチャットUI
- メッセージ送信 → バックエンドAPIへPOST
- レスポンスをチャット画面に表示
- ローディングアニメーション表示

### 3.2 バックエンド
- Node.js（Express）または Python（FastAPI）
- 役割：
  - フロントからのメッセージを受信
  - LLM API にリクエスト送信
  - レスポンスをフロントに返却
  - APIキーを安全に保持（環境変数）

### 3.3 LLM API
- OpenAI API / Azure OpenAI / Gemini API のいずれか  
- モデルは任意（gpt-4o-mini / gpt-4.1 / Gemini Flash など）

---

## 4. 機能要件

### 4.1 必須機能
- **チャット送受信**
- **ChatGPT風UI**
- **ログイン不要**
- **APIキー非公開（バックエンドのみ保持）**
- **メッセージ履歴の画面内保持**
- **エラーハンドリング（APIエラー時の通知）**

### 4.2 任意機能（拡張）
- 授業テーマ別の「おすすめプロンプト」表示
- 会話ログのサーバー保存（授業分析用）
- 新規チャット開始ボタン
- ダークモード
- モデル切り替え（管理者のみ）

---

## 5. 非機能要件
- **セキュリティ**
  - APIキーは `.env` に保存
  - CORSでアクセス元を制限（学校LAN内のみ）
  - HTTPS推奨

- **パフォーマンス**
  - レスポンスはストリーミング対応（任意）
  - 1秒以内のUI応答（ローディング表示含む）

- **可用性**
  - 授業時間中に安定稼働すること

---

## 6. 画面仕様（UI）

### 6.1 メイン画面構成
- 左サイドバー
  - 「新しいチャット」
  - 授業テーマ別プロンプト（任意）
- メインチャット画面
  - メッセージバブル（ユーザー右 / AI左）
  - ローディングアニメーション
  - スクロール自動追従
- 入力欄
  - textarea
  - 送信ボタン（紙飛行機アイコン）

### 6.2 デザイン要素
- 背景：#f7f7f8（ChatGPT風）
- フォント：Inter / Noto Sans
- メッセージバブル：
  - ユーザー：右寄せ・青系
  - AI：左寄せ・白背景・灰枠

---

## 7. API仕様（バックエンド）

### 7.1 エンドポイント
#### POST `/api/chat`
**リクエスト**
```json
{
  "message": "こんにちは、AIとは何ですか？"
}
```

**レスポンス**
```json
{
  "reply": "AIとは人工知能のことで…"
}
```

### 7.2 エラーレスポンス
```json
{
  "error": "API request failed"
}
```

---

## 8. バックエンド実装方針

### 8.1 Node.js（Express）例
- `server.js`
- `dotenv` でAPIキー管理
- `fetch` でLLM APIを呼び出す

### 8.2 Python（FastAPI）例
- `main.py`
- `pydantic` で型定義
- `httpx` でAPI呼び出し

## 10. 運用方法
